# Web Design Engineer Delivery Card

## Use

Use for visual Web/UI pages, dashboards, prototypes, workbench screens, animations, data visualization, and design-system exploration.

## Required Inputs

- Product goal and product information architecture.
- Existing UI/code or design references.
- Target user and primary workflow.
- Responsive and accessibility constraints.

## Required Outputs

- Layout and visual hierarchy.
- Component and interaction behavior.
- State coverage: loading, success, empty, error, permission, blocked.
- Responsive behavior and implementation notes.

## Stop

- Product information architecture is missing.
- Main UI would expose internal fields as user-facing content.
- Visual direction cannot be inferred and no reference is available.
