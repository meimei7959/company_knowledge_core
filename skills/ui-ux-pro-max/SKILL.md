# 前端设计质量标准

> 运营成功 > 功能完整。每个功能必须回答：它帮助获取用户？留住用户？还是提升交付质量？

## Purpose

为前端页面、工作台、仪表盘、交互流程和组件设计提供体验质量标准，确保界面可理解、可操作、可反馈、可响应。

## Triggers

- 设计 Agent 需要做 UI/交互设计、视觉评审、组件状态设计、前端体验质量门禁。
- 研发 Agent 要实现用户可见页面，并需要明确加载、成功、错误、空状态和操作反馈。
- 测试 Agent 要验证页面是否体验友好、交互完整、中文可理解。

## Inputs

- 产品经理给出的需求、信息架构、用户流程、验收标准。
- 设计稿、现有页面、组件库、视觉规范、前端实现约束。
- 页面状态、权限状态、异常状态、响应式要求。

## Workflow

1. 先确认页面目标、用户、核心操作和成功标准。
2. 检查四态：加载、成功、错误、空。
3. 检查操作反馈：点击、loading、成功/失败、撤销或恢复。
4. 检查视觉系统：颜色、间距、阴影、密度、动效、响应式。
5. 输出设计/研发/测试可执行清单。

## Outputs

- UI/UX 质量检查结论。
- 页面状态和交互反馈要求。
- 视觉规范、动效规范、响应式要求。
- 研发和测试验收清单。

## Quality Gate

- 页面能让普通用户理解当前状态、下一步动作、错误原因和责任人。
- 所有用户操作有反馈。
- 组件覆盖加载、成功、错误、空状态。
- 移动端和窄屏有合理布局。
- 主界面不暴露内部字段或工程黑话。

## Failure Routes

- 缺产品信息架构：退回产品经理 Agent。
- 缺视觉/交互设计：退回设计 Agent。
- 实现未覆盖四态或反馈：退回研发 Agent。
- 测试发现用户不可理解：退回设计或研发 Agent。

## 工具链

| 工具 | 用途 | 安装 |
|------|------|------|
| **Framer Motion** | 动画（页面过渡、列表排序、退出、手势） | `npm i framer-motion` |
| **auto-animate** | 一行代码给列表/DOM 变化加过渡 | `npm i @formkit/auto-animate` |
| **cmdk** | 命令面板（Cmd+K） | `npm i cmdk` |
| **Sonner** | Toast 通知（堆叠、可撤销） | `npm i sonner` |
| **Lucide** | 图标（700+，tree-shakable） | `npm i lucide-react` |

新页面可用 shadcn/ui + Tailwind 替代 Ant Design，旧页面保持不动，两套共存无冲突。

## 必须实现的模式

| 模式 | 规则 |
|------|------|
| **四态完整** | 每个组件必须覆盖：加载态（骨架屏） / 成功态 / 错误态 / 空态 |
| **骨架屏替代 Spinner** | 用内容形状占位，不用转圈 |
| **空状态有引导** | 永远不显示白板，用文字说明 + 行动按钮 |
| **操作必有反馈** | 点击 → loading → 成功/失败 Toast，禁止点了没反应 |
| **乐观更新** | UI 不等服务器确认就先变，失败时回滚 |
| **列表有动效** | 增删排序必须有过渡动画（auto-animate 或 Framer Motion） |
| **页面有过渡** | 路由切换用 Framer Motion layout 动画，禁止硬切 |
| **hover/active 反馈** | 所有可交互元素必须有视觉状态变化 |
| **渐进式披露** | 先显示摘要，按需展开。次要操作折叠进菜单 |
| **可撤销 > 二次确认** | 删除/归档用 Undo Toast，不用"确定要删除吗？"弹窗 |

## 动效时间

| 场景 | 时长 | 缓动 |
|------|------|------|
| hover/active/toggle | < 150ms | ease-out |
| 展开/折叠/页面切换 | 200-300ms | cubic-bezier(0.16, 1, 0.3, 1) |
| 入场/骨架屏消失 | 300-400ms | cubic-bezier(0.34, 1.56, 0.64, 1) |

**动画永远不阻塞交互。**

## 提交检查清单

每次前端改动，必须逐项确认：

**功能**
- [ ] 四态覆盖：加载 / 成功 / 错误 / 空
- [ ] 操作有反馈：loading + 结果 Toast
- [ ] 响应式：Mobile 有合理降级

**视觉**
- [ ] 颜色用 CSS 变量（`--color-*`），无硬编码 hex
- [ ] 间距遵循 8px 网格（`--space-*`）
- [ ] 阴影用系统预设（`--shadow-*`）

**交互**
- [ ] 列表变化有过渡动画
- [ ] hover/active 有视觉反馈
- [ ] Enter 确认、Esc 取消

**代码**
- [ ] 样式用 CSS Modules 或 Tailwind，无 inline style（动态值除外）
- [ ] 组件放对目录（ui/ / common/ / 业务模块/）
- [ ] 2 处以上引用已抽公共组件
