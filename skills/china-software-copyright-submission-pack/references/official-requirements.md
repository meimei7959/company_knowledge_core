# 当前规则基线与核验范围

规则核验日期：2026-06-22。

本文件用于说明 Skill 的工程校验基线，不替代中国版权保护中心提交当日的登记指南、在线表单或补正通知。

## 1. 法定材料框架

依据《计算机软件著作权登记办法》，申请软件著作权登记的材料框架包括：

1. 按要求填写的软件著作权登记申请表；
2. 软件的鉴别材料；
3. 相关证明文件。

相关证明文件包括申请人身份证明；在相应情形下，还包括著作权归属合同或项目任务书、原软件著作权人的许可证明、权利继承/受让/承受证明等。

国家版权局文字版：

- https://www.ncac.gov.cn/xxfb/flfg/bmgz/202410/P020241015604759474834.docx

## 2. 鉴别材料基线

一般交存工程基线：

- 鉴别材料包括源程序和任意一种软件文档；
- 源程序和文档通常分别提交前、后各连续 30 页；不足 60 页时全部提交；
- 除特定情况外，源程序每页不少于 50 行；文档每页不少于 30 行；
- 源程序和文档页眉标注的软件名称、版本号应与申请表一致；
- A4 规格，按当前登记系统要求上传或提交。

Skill 默认只支持一般交存。例外交存、封存、代码遮盖或商业秘密处理必须停止自动最终化并转人工处理。

## 3. 2026 年申请表变化

深圳市软件行业协会于 2026-03-16 转发的《关于共同维护计算机软件版权登记良好生态的通知》显示，新版申请表包含更严格的诚信和 AI 使用声明，并要求经办人签名及身份证号码。公开转载的信息还显示“主要功能和技术特点”填写范围有调整。

- https://www.ssia.org.cn/page59?article_id=2588

由于在线表单、字段说明和执行口径可能变化，提交前必须由申请负责人打开当前登记系统重新核对。Skill 不缓存官方表单，也不代替申请人作出声明。

## 4. 工程边界

Skill 可以机械执行：

- 读取和枚举真实第一方源码；
- 固定 Git 快照；
- 排除依赖、生成文件和敏感信息；
- 按固定规则分页；
- 校验路径、名称、版本和页数；
- 复制原始公司材料；
- 生成内部目录、清单、哈希、缺件表和工作 ZIP。

Skill 不可以：

- 创作拟申报代码或文档正文；
- 生成官方申请表或签章件；
- 推断权属、日期和发表事实；
- 对 AI 声明作法律解释；
- 保证受理或登记结果。
