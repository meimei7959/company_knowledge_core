# 软著材料交付卡

## 主责

知识工程 Agent。

## 触发方

项目经理 Agent 在 PRD 定稿、项目计划、Release Candidate、上线验收或项目关闭前触发。

## 协作输入

- 产品经理 Agent：软件名称候选、产品定位、功能边界、主要功能清单。
- 研发 Agent：冻结 commit、源码范围、构建命令、依赖说明。
- 测试 Agent：测试报告、验收记录、真实运行截图。
- 申请负责人：申请表、主体证明、权属证明、AI 使用事实确认。

## Agent 可做

- 初始化材料目录。
- 收集、检查、映射、归档材料。
- 生成工作资料包和缺件清单。
- 生成内部复核材料和一一对应表。

## Agent 不可做

- 代写拟申报代码、说明书正文、主要功能描述、功能说明或操作步骤。
- 伪造申请表、签章、营业执照、合同、授权或权属事实。
- 替申请负责人决定 AI 使用声明。
- 承诺登记一定通过。

## 完成标志

- `SUBMISSION_READY`：材料齐全且人工最终化通过；仍需申请负责人按当前系统提交。
- `READY_FOR_HUMAN_COMPLETION`：工作包完成但缺人工材料或确认。
- `INTERNAL_ONLY_AI_USE`：AI 使用事实与当前公开声明存在冲突，只保留内部资料。
- `BLOCKED`：构建、敏感信息、路径、材料一致性或权属材料存在阻断。
