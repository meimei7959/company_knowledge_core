# 公司、申请人和权属材料规范

## 1. 基本原则

这些材料不由 Skill 生成，只能由申请人、公司、相关权利人或代理方提供。Skill 的作用是：

- 验证配置路径是否存在；
- 按申报项复制到固定槽位；
- 计算文件哈希；
- 在映射表中标记 `PRESENT` 或 `MISSING`；
- 在正式 ZIP 中排除 README、交接单和内部资料。

## 2. 正式申请表

- 必须来自提交当日的当前版权登记系统；
- 由申请人按当前说明填写；
- 完成当前要求的声明、经办人信息、签名或盖章；
- Skill 生成的字段交接单不能代替正式表。

## 3. 申请人身份证明

企业申请时通常配置 `business_license`；自然人或其他主体配置 `identity_document`。提交前人工检查：

- 主体名称与申请表、著作权人一致；
- 文件完整、清晰、在有效状态；
- 签章和复印件标注符合当前渠道要求；
- 敏感身份文件存放在受控目录，不进入公开仓库。

## 4. 权属材料

- 合作开发：合作开发协议及权利归属约定；
- 委托开发：委托开发合同及权利归属约定；
- 下达任务开发：项目任务书或相应证明；
- 二次开发：原软件著作权人许可证明；
- 继承/受让/承受：相应权利转移证明；
- 代理办理：当前渠道要求的委托材料；
- 外文证明：符合当前要求的中文译本。

Skill 不判断合同法律效力，也不补写缺失条款。
