# 输出契约

## 1. 申报总览

- `00_申报材料一一对应表.md/.csv`：F-01 至 F-10 的适用性、来源、状态、目标和实际文件；
- `00_材料状态.json`：机器可读状态、AI 门、Git 快照、缺件和警告；
- `00_提交顺序与操作说明.md`：人工从工作包到实际上传的步骤；
- `00_缺件清单.md`：只列当前适用但缺失的正式项和未完成确认。

## 2. 正式上报材料

### 01 软件著作权登记申请表

- 正式签章件；
- 字段交接单仅为工作文件，会被正式 ZIP 排除。

### 02 软件鉴别材料

- `02-1_源程序鉴别材料.docx/.pdf`；
- `02-2_用户操作说明书.docx/.pdf` 或设计说明书版本。

### 03 申请人身份证明

- 企业营业执照或其他主体身份证明。

### 04 权属及其他证明（按需）

- 合同/任务书；
- 原软件许可；
- 继承/受让/承受证明；
- 代理委托；
- 外文译本；
- 系统或补正要求的其他材料。

## 3. 内部复核佐证

- 原始截图；
- 界面截图册 DOCX/PDF；
- 功能模块说明书 DOCX/PDF；
- 功能—代码—截图映射 CSV/JSON；
- 源码清单、软件信息草稿、依赖/IP 报告；
- 一致性报告、人工复核清单和构建日志。

## 4. ZIP

### 工作资料包

始终生成，包含总览、正式候选区、内部佐证、缺件区和合规记录。它不是“可以直接整体上传”的文件。

### 正式上报包

仅在状态为 `SUBMISSION_READY` 时生成。只包含：

- 实际签章申请表；
- 源程序鉴别材料；
- 文档鉴别材料；
- 主体身份证明；
- 当前适用权属及其他证明；
- ZIP 内部清单和哈希。

不包含工作交接单、README、截图册、功能模块独立说明、内部报告或日志。
