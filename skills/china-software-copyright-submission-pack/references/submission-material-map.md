# 申报项与输出文件映射

## 1. 核心正式申报项

| 编号 | 申报项 | 适用性 | 状态判定 | 正式输出文件 | 不足时动作 |
|---|---|---|---|---|---|
| F-01 | 软件著作权登记申请表 | 必需 | `company-materials.json.official_application_form` 指向实际签章文件 | `01_正式上报材料/01_软件著作权登记申请表/01_正式申请表_签章件.<ext>` | 在当前系统人工填报、下载、声明、签章后配置路径 |
| F-02 | 源程序鉴别材料 | 必需 | DOCX 和 PDF 均成功生成且人工合规门通过 | `02-1_源程序鉴别材料.docx/.pdf` | 修复源码范围、敏感信息、分页或 PDF 转换问题 |
| F-03 | 文档鉴别材料 | 必需 | DOCX 和提交版 PDF 均成功生成且人工合规门通过 | `02-2_用户操作说明书.docx/.pdf` 或 `02-2_软件设计说明书.docx/.pdf` | 完成人工原创正文、截图、路径和一致性复核 |
| F-04 | 申请人身份证明 | 必需 | 企业有营业执照；其他申请人有相应身份证明 | `03_申请人身份证明/03-1_...` | 由申请人提供清晰有效文件 |

## 2. 条件材料判定

| 编号 | 当前适用条件 | 配置来源 |
|---|---|---|
| F-05 | 合作开发、委托开发、下达任务开发，或明确存在归属合同/任务书 | `application.development_mode`、`application.has_ownership_contract_or_task`、`company-materials.json.ownership_scenario` |
| F-06 | 当前软件基于原软件修改或二次开发 | `application.based_on_original_software` 或 `company-materials.json.modified_from_other_software` |
| F-07 | 权利取得方式不是原始取得，或存在继承/受让/承受情形 | `application.rights_acquisition` 或 `company-materials.json.rights_succession` |
| F-08 | 委托代理办理 | `application.uses_registration_agent` |
| F-09 | 证明文件含外文材料 | `application.has_foreign_documents` |
| F-10 | 当前登记系统或补正通知另有要求 | `company-materials.json.other_documents`，人工判断 |

条件材料的自动判定只是缺件提示。提交前仍必须由申请负责人按真实权属和当前系统要求确认。

## 3. 工作文件与正式文件的区分

| 文件 | 默认上传？ | 用途 |
|---|---|---|
| 申请表字段交接单 | 否 | 人工把已确认字段录入官网 |
| 原始界面截图 | 否 | 证实说明书图片来自真实版本 |
| 界面截图册 | 否 | 内部逐图复核、隐私检查和补正准备 |
| 功能模块说明书 | 否 | 内部核验功能、代码和截图的覆盖关系 |
| 功能—代码—截图对应表 | 否 | 追溯证据、定位缺失或不一致 |
| 源码文件清单 | 否 | 证明代码选择过程可复现 |
| 依赖/IP 报告 | 否 | 避免第三方代码误作第一方材料 |
| 一致性报告 | 否 | 人工签章和上传前检查 |
| F-01 至 F-09 实际材料 | 是或适用时是 | 正式登记系统材料槽位 |

只有登记系统或补正通知明确要求时，才把内部佐证单独转为正式补充材料；转入前须人工检查是否与 F-02/F-03 完全一致。
