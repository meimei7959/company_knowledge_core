# 文档鉴别材料与截图规范

## 1. 文档类型

- 移动 App、Web、桌面应用：通常使用用户操作说明书；
- API、算法服务、后台服务、组件或无界面软件：可使用软件设计说明书；
- 最终类型以申请人选择和当前系统要求为准。

## 2. 正式说明书最小结构

### 用户操作说明书

1. 软件概述；
2. 运行环境；
3. 安装、启动和登录；
4. 功能模块结构；
5. 各功能的前置条件、操作步骤、结果和真实截图；
6. 异常处理；
7. 退出、权限和安全相关操作；
8. 版本说明。

### 软件设计说明书

1. 软件概述和设计目标；
2. 系统架构；
3. 运行环境和依赖；
4. 模块划分；
5. 数据流、接口和关键流程；
6. 异常及安全设计；
7. 部署和运行；
8. 版本说明。

## 3. 一致性

- 页眉中的软件全称、版本号与申请表一致；
- 说明书只描述 `verified` 功能；
- 功能名称、代码路径、截图和步骤可在证据映射中追溯；
- 截图来自当前冻结版本；
- 图中的 App 名称、版本、菜单和功能与正文一致；
- 不含真实用户手机号、身份证号、令牌、密钥或生产数据。

## 4. 截图输出

同一张截图可能同时出现在：

1. 正式说明书的相应功能章节；
2. 内部原图目录；
3. 内部界面截图册；
4. 功能—代码—截图对应表。

内部截图册和独立功能模块说明书默认不单独上传。它们用于人工复核和补正准备。

## 5. 页数和排版

Skill 按一般交存基线将文档提交版控制为：

- 不足 60 页：全部；
- 超过 60 页：前 30 页＋后 30 页；
- 页眉显示软件全称和版本；
- 页脚或页码区域显示连续页码；
- A4 版面；
- PDF 生成后应由人逐页视觉检查。
