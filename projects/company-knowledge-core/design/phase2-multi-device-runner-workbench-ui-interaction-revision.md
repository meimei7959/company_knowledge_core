# 阶段二同事接入工作台 UI 与交互设计返工

## 设计结论

本返工稿引用并遵守 `projects/company-knowledge-core/workflows/phase2-multi-device-collaboration-orchestrator-orchestrator.md`：阶段二工作台是用户可读的协作设备控制台，不是 Runner 调试面板。设计 Agent 的主责是 UI 设计、交互设计、视觉层级、中文文案、组件状态和可交付标注；信息架构只作为辅助结构。

阶段二新增入口名为“协作设备”。用户第一眼应看到：

- 当前项目是否允许同事电脑加入。
- 现在有几台电脑可接任务、几台异常、几项等待授权。
- 邀请同事、确认授权、转交任务、查看证据的主路径。
- 任务为什么路由给某台电脑，卡点是谁处理。

主界面不得把 `runnerId`、`deviceId`、`leaseId`、`claimId`、raw status、路径、token、endpoint、错误堆栈作为标题、主列、按钮或错误主文案。它们只进入“技术详情/证据”折叠区。

## 视觉层级

- 页面标题：`协作设备`，副标题：`邀请同事电脑加入当前项目，并查看任务由哪台电脑执行。`
- 顶部主按钮只有一个：`邀请同事`。
- 状态标签用短词：`可接任务`、`执行中`、`等待授权`、`离线`、`需处理`。
- 风险标签只跟随对象：`权限不足`、`授权将过期`、`只读`。
- 操作按钮优先级：主动作 > 次动作 > 更多菜单 > 技术详情。
- 颜色：绿色表示可用，蓝色表示执行中，黄色表示等待/授权，红色表示失败/阻塞，灰色表示离线/只读。页面不得被单一色系主导。

## 邀请同事流程

### 入口

- 顶部主按钮：`邀请同事`。
- 空状态主按钮：`邀请同事接入电脑`。
- 任务卡片按钮：`邀请可执行同事`，携带当前任务类型上下文。

### 弹窗布局

```txt
邀请同事电脑加入项目                         [关闭]

项目
真知公司知识核心

允许执行的工作
[ ] 设计  [ ] 开发  [ ] 测试  [ ] 资料整理  [ ] 项目管理

资料与代码范围
( ) 只读项目资料
( ) 项目资料 + 指定代码仓库
( ) 指定任务临时授权

邀请有效期
[1 小时] [24 小时] [7 天]

审批人
项目负责人 / 管理员

提示：对方接受后，还需要完成电脑配对和授权确认，才会开始接任务。

[取消] [生成邀请]
```

生成后显示：

- 主文案：`邀请已生成，等待同事接受。`
- 次信息：`有效期至今天 20:20。邀请不会授予超出当前项目的资料、仓库或工具权限。`
- 操作：`复制邀请链接`、`发送到飞书`、`撤销邀请`。
- 技术详情折叠：`pairingRequestRef`、创建审计引用、过期时间原始值。

失败态：

| 场景 | 标题 | 正文 | 动作 |
| --- | --- | --- | --- |
| 无权限生成 | 无权限邀请同事 | 你可以查看协作设备状态，但不能创建接入邀请。 | 申请权限 |
| 只读降级 | 当前为只读模式 | 系统无法确认写入权限，暂时不能生成邀请。 | 重新连接 |
| 网络失败 | 邀请没有生成成功 | 项目状态没有改变，请重试。 | 重试 |
| 审批人缺失 | 需要先指定审批人 | 接入同事电脑需要项目负责人或管理员确认。 | 选择审批人 |

## 配对授权流程

### 主路径

```txt
项目 Owner 生成邀请
-> 同事打开邀请并提交电脑名称
-> 工作台显示“同事电脑请求接入”
-> Owner 查看申请
-> 选择授权范围
-> 确认授权
-> 设备进入“可接任务”
```

### 配对申请卡

```txt
同事电脑请求接入
张三的 MacBook Pro 想加入“真知公司知识核心”

可做工作：开发、测试
申请资料：项目资料、指定代码仓库
风险提示：代码仓库访问需要负责人确认

[拒绝] [查看详情] [同意接入]
```

### 授权抽屉

分区：

1. 申请概况：申请人、电脑名称、项目、申请时间。
2. 可做工作：设计、开发、测试、资料整理、项目管理。
3. 资料范围：项目资料、指定 SourceMaterial、只读知识。
4. 代码范围：无代码访问、指定仓库只读、指定仓库读写。
5. 工具范围：仅已批准工具，默认最小权限。
6. 有效期和撤销：1 天、7 天、长期但可撤销。
7. 技术详情/证据：默认折叠。

主按钮文案：

- `确认授权`
- `拒绝接入`
- `保存为只读设备`

确认前二次提示：

```txt
确认让张三的 MacBook Pro 加入项目？
这台电脑可以处理：开发、测试。
可以读取：项目资料、指定代码仓库。
授权后仍会经过任务路由和租约校验，不会自动执行未分配任务。

[返回修改] [确认授权]
```

张三 · macOS             当前任务：暂无任务
授权：当前项目 + 指定代码仓库       最近在线：刚刚

| 等待授权 | 黄色标签，授权申请置顶 | 确认授权 | 拒绝 | 缺少资料/仓库/工具授权。 |
| 离线 | 灰色标签，最近在线突出 | 转交任务 | 等待恢复 | 离线设备不接新任务。 |
| 需处理 | 红色标签，显示阻塞原因 | 查看原因 | 转交/重试 | 需要项目负责人或管理员处理。 |

| 授权将过期 | 黄色风险标签 | 续期授权 | 撤销 | 只影响新任务，已有租约按策略处理。 |
| 已撤销 | 灰色标签，低优先级 | 查看记录 | 重新授权 | 不显示为可选执行设备。 |

## 任务路由状态

右侧 RouteBoard 分组：

- `待分配`
- `等待授权`
- `执行中`
- `等待回传`
- `待验收`
- `需人工处理`

任务条结构：

```txt
阶段二：同事接入工作台 UI 与交互设计返工
状态：执行中
分配给：本机 Codex 工作台
为什么：具备设计交付能力，当前项目已授权，设备在线
卡点：无卡点
下一步：提交 TaskResult 后交项目经理验收
[查看任务] [查看证据]
```

状态映射：

| 用户状态 | 内部来源 | 主提示 | 可用动作 |
| --- | --- | --- | --- |
| 待分配 | pending / waiting_runner | 正在匹配可执行同事电脑。 | 查看条件、邀请同事。 |
| 可执行 | eligible runner found | 可由张三的 Mac 执行。 | 指派、稍后执行。 |
| 等待授权 | blocked + approval | 需要李四确认代码仓库访问。 | 发起授权、换设备。 |
| 执行中 | processing + fresh lease | 张三的 Mac 正在处理任务。 | 查看进度、查看证据。 |
| 等待回传 | writeback retry | 正在同步结果和证据。 | 重试同步、查看已回传部分。 |
| 待验收 | waiting_acceptance | 结果已写回，等待复核。 | 去验收、查看证据。 |
| 需人工处理 | stale lease / conflict | 任务需要人工确认后继续。 | 查看原因、转交、取消。 |
| 已完成 | done | 已生成结果和证据。 | 查看证据。 |
| 需修改 | changes_requested / rejected | 结果未通过，需要返工。 | 创建返工、查看意见。 |

## 弹窗、抽屉、菜单

| 交互面 | 打开方式 | 内容 | 关闭/提交规则 |
| --- | --- | --- | --- |
| 邀请弹窗 | `邀请同事` | 项目、工作类型、资料/代码范围、有效期、审批人 | 生成前可关闭；生成后撤销需确认。 |
| 授权抽屉 | 授权申请 `同意接入` | 申请概况、授权范围、风险、技术详情 | 确认授权必须二次确认。 |
| 设备详情抽屉 | 设备行 `查看` | 概况、当前任务、授权、执行记录、证据 | 抽屉内写操作仍走权限 gate。 |
| 转交任务弹窗 | 任务 `转交` | 当前设备、候选设备、转交原因 | 只能选择合格设备；写审计。 |
| 暂停接单确认 | 设备 `暂停接单` | 影响范围、当前任务处理策略 | 若有执行中任务，必须选择等待完成/转交/取消。 |
| 技术详情折叠 | `查看技术详情` | 内部引用、原始状态、脱敏错误、审计 | 默认收起；普通用户不需要展开也能完成流程。 |
| 更多菜单 | 行尾 `...` | 复制说明、查看记录、撤销授权、重新授权 | 危险动作放菜单底部并二次确认。 |

## 异常、权限、空、加载、只读

| 状态 | 页面表现 | 中文文案 | 动作 |
| --- | --- | --- | --- |
| 首次空 | 主内容空状态，不显示空表格 | 还没有同事电脑加入这个项目 | 邀请同事 |
| 无可执行设备 | 任务路由黄色提示 | 当前没有能处理此类任务的设备 | 邀请可执行同事 |
| 加载中 | 骨架屏保留总览/列表结构 | 正在读取项目协作设备 | 超过 8 秒显示重试 |
| 局部加载 | 行内 shimmer，不清空旧数据 | 正在更新状态 | 无 |
| 加载失败 | 顶部错误条 + 旧数据灰化 | 暂时无法更新设备状态 | 重试 |
| 无项目权限 | 顶部只读条 + 写按钮禁用 | 你暂无此项目的协作设备权限 | 申请权限 |
| 只读降级 | 黄色提示条 | 当前为只读模式，只展示最近一次同步结果 | 重新连接 |
| 数据过期 | 时间提示 | 状态可能不是最新，最近更新于 10 分钟前 | 刷新 |
| 邀请过期 | 弹窗失败态 | 邀请已过期，请重新生成 | 重新生成邀请 |
| 设备离线 | 设备行灰态 + 任务提示 | 同事电脑暂时离线，可等待恢复或转交 | 转交任务 |
| 权限不足 | 授权提示条 | 这台电脑缺少必要授权 | 发起授权 |
| 写回失败 | 任务路由状态 | 结果暂未同步完成 | 重试同步 |
| 越权拒绝 | 任务/设备错误 | 这台电脑不能读取当前任务所需资料 | 换设备 / 发起授权 |

禁用按钮必须有 hover/focus 原因：

- `无权限邀请同事`
- `只读模式下不能暂停设备`
- `该设备离线，不能接新任务`
- `正在执行任务，不能直接撤销授权`

## 研发标注

1. read model 需要直接提供可读字段：`displayName`、`ownerLabel`、`availabilityLabel`、`workTypeLabels`、`authorizationSummary`、`currentTaskLabel`、`lastSeenLabel`、`riskLabels`、`primaryAction`。
2. 前端可保留原始字段用于技术详情，但主模板只能渲染可读字段。
3. 所有写操作按钮必须绑定服务端 permission gate，失败返回 `displayMessage`。
4. `TechnicalDetails` 默认折叠，并对 token、secret、路径、错误堆栈脱敏。
5. `RouteBoard` 每条记录必须有 `assignedDeviceLabel`、`routeReason`、`blockerLabel`、`nextOwnerLabel`、`nextAction`。
6. 所有状态映射都要覆盖未知值：未知状态显示“需处理”，并进入技术详情，不直接渲染 raw status。
7. 空/加载/失败/只读状态不能清空已知安全数据；加载失败保留最近同步结果。
8. 窄屏用卡片流，不复用横向表格造成文字截断。
9. 危险动作必须二次确认：撤销授权、暂停接单、取消任务、转交执行中任务。
10. 每个写操作完成后必须能在操作记录或审计入口看到可读摘要。

## 测试验收项

### UI 可读性

- [ ] 主导航显示“协作设备”，不以 Runner 作为普通用户入口名。
- [ ] 设备列表第一列是同事电脑中文名；没有中文名时显示“未命名的同事电脑”。
- [ ] 主界面不出现 `runnerId`、`deviceId`、`leaseId`、`claimId`、raw status、路径、token、endpoint。
- [ ] 所有按钮为中文动词短语。
- [ ] 所有错误提示包含业务影响、原因、下一步。

### 邀请与授权

- [ ] 有权限用户可以打开邀请弹窗，生成邀请后看到复制和发送入口。
- [ ] 无权限、只读、网络失败、审批人缺失都有独立文案。
- [ ] 配对申请卡显示申请人、电脑名称、项目、可做工作、资料/代码范围。
- [ ] 授权抽屉显示项目、工作、资料、代码、工具、有效期、撤销入口。
- [ ] 授权确认有二次确认。

### 设备与路由

- [ ] 可接任务、执行中、等待授权、离线、需处理、只读、授权将过期、已撤销均有视觉状态。
- [ ] 执行中设备不能直接撤销授权，必须先选择处理当前任务。
- [ ] RouteBoard 每条任务显示分配给谁、为什么、卡点、下一步谁处理。
- [ ] waiting_runner、stale lease、runner offline、writeback failed、needs approval 都映射为中文状态。

### 降级与窄屏

- [ ] 首次空状态、无可执行设备、加载中、局部加载、加载失败、只读降级、数据过期均可见。
- [ ] 禁用按钮 hover/focus 显示中文原因。
- [ ] 768-1199px 单列可完成邀请、授权、查看路由。
- [ ] <768px 保留项目选择器、邀请入口、设备状态、任务路由；主动作不截断。

### 证据与安全

- [ ] 技术详情默认折叠。
- [ ] 技术详情展开后只显示脱敏值和审计引用。
- [ ] 邀请 token、secret、密码不在任何页面出现。
- [ ] 写操作后有操作记录或审计入口。
- [ ] 结果写回状态可从任务路由进入“查看执行证据”。
