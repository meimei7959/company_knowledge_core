# 阶段二方案二中枢工作台 UI 与交互设计

## 设计结论

工作台定位从“纯只读监管”调整为“入口可操作，执行监管只读”。

- 入口操作区允许用户发起三类受控动作：创建项目、邀请/注册电脑、注册工具。
- 入口动作只提交申请或生成受控接入材料，不直接绕过权限、审批、确认和审计。
- 运行监管区只展示中枢已经记录的事实：项目、电脑、任务、Agent、模型、token 消耗、工具、异常和证据。
- 运行监管区不提供派单、改状态、禁用电脑、验收通过、重试执行、释放占用等改变运行状态的按钮。
- 主界面使用中文名称、负责人、任务标题、需求标题和业务状态；内部英文 ID 只能放入默认收起的“技术详情”，且需要脱敏。

一句话给用户看的定位文案：

> 这里可以申请接入项目、电脑和工具，也可以查看所有电脑正在为哪些项目工作；运行状态只展示中枢记录，不在这里直接改任务。

## 页面框架

```txt
┌──────────────────────────────────────────────────────────────────────────────┐
│ 中枢工作台                                                                  │
│ 项目选择：全部项目 / 真知公司知识核心     数据来自：线上中枢     最近同步：刚刚 │
│ [创建项目] [邀请/注册电脑] [注册工具] [刷新]                                  │
├──────────────┬──────────────────────────────────────────────┬────────────────┤
│ 左侧导航      │ 主内容                                       │ 右侧详情/证据   │
│ 总览          │ 总览指标                                     │ 当前选中项目     │
│ 项目          │ 项目列表 / 项目任务明细                       │ 当前选中电脑     │
│ 电脑          │ 电脑列表 / 电脑详情                           │ 当前选中任务     │
│ 工具          │ 工具登记状态 / 工具使用情况                    │ 异常说明         │
│ 任务          │ 任务与需求映射                                │ 审批和审计       │
│ 审批与审计    │ 异常和待处理                                  │ 技术详情         │
└──────────────┴──────────────────────────────────────────────┴────────────────┘
```

### 顶部项目选择

项目选择是所有监管信息的统一上下文。

| 状态 | 展示方式 | 用户能理解的问题 |
| --- | --- | --- |
| 全部项目 | 下拉第一项，文案为“全部项目” | 当前中枢一共有多少项目、多少电脑在工作。 |
| 单个项目 | 显示项目中文名、负责人、协作状态 | 这个项目有哪些任务、哪些电脑参与、哪里异常。 |
| 无项目 | 显示“还没有项目” | 可以从“创建项目”开始。 |
| 只看权限内项目 | 下拉底部说明“只显示你有权查看的项目” | 为什么看不到某些项目。 |
| 数据过期 | 顶部黄提示“中枢状态可能不是最新” | 当前数据能看，但不能当作实时事实。 |

不要把 `projectId`、仓库路径或内部项目代号作为主文案。需要排障时，在技术详情里显示“项目技术标识，已脱敏”。

## 操作边界

### 入口可操作

入口操作只用于申请和登记，不用于改运行状态。

| 入口 | 主按钮文案 | 成功后的用户反馈 | 审计要求 |
| --- | --- | --- | --- |
| 创建项目 | 创建项目 | 已提交项目创建申请，等待项目负责人确认。 | 记录申请人、项目名称、资料范围、审批人、提交时间。 |
| 邀请/注册电脑 | 邀请/注册电脑 | 已生成电脑接入申请，电脑通过审核后才会开始上报。 | 记录申请人、电脑负责人、项目范围、可做工作、审批结论。 |
| 注册工具 | 注册工具 | 已提交工具登记申请，通过后才会出现在可用工具里。 | 记录工具名称、用途、负责人、适用项目、权限范围、风险说明。 |

### 执行监管只读

运行监管区域只允许：

- 查看项目、电脑、任务、Agent、模型、token 消耗、工具和异常。
- 筛选、排序、搜索、展开详情。
- 复制用户可读摘要。
- 打开证据、审批记录和审计记录。
- 查看脱敏技术详情。

运行监管区域不允许：

- 直接创建、派发、取消、转派或验收任务。
- 直接启停、禁用、释放或重试电脑执行器。
- 直接修改 Agent 状态、模型选择、token 数字或工具授权。
- 直接写入 TaskResult、AgentRun、审计或通知。

禁用文案不能只写“不可用”。必须说明原因，例如：

- “运行监管区只读，不能在这里改任务状态。”
- “电脑注册需要负责人审批，通过后才会接入。”
- “工具会访问项目资料，需要先完成工具负责人确认。”

## 首屏总览

首屏需要让普通用户在 10 秒内判断：有几个项目、几台电脑、哪些任务在跑、哪里异常、哪些入口需要处理。

### 总览指标

用紧凑指标，不做夸张大卡。建议 6 个指标：

| 指标 | 示例文案 | 点击后 |
| --- | --- | --- |
| 项目 | 4 个项目 | 进入项目列表，按最近更新排序。 |
| 电脑 | 6 台电脑接入 | 进入电脑列表，优先显示异常和执行中。 |
| 运行中任务 | 9 项任务运行中 | 进入任务监管，筛选“运行中”。 |
| 异常电脑 | 1 台电脑异常 | 进入异常列表，选中异常电脑。 |
| 待验收任务 | 3 项待验收 | 进入任务列表，筛选“待验收”。 |
| 待审批入口 | 2 个申请待审批 | 进入审批与审计，筛选入口申请。 |

### 总览提示条

总览下方放一条状态提示。按严重程度只显示一条主提示：

| 场景 | 标题 | 正文 | 操作 |
| --- | --- | --- | --- |
| 正常 | 中枢运行正常 | 多台电脑正在按项目上报状态，任务结果以中枢记录为准。 | 查看任务 |
| 有异常 | 1 台电脑需要处理 | 李雷的 Mac 已 18 分钟未上报，正在影响 1 项任务。 | 查看异常 |
| 有审批 | 2 个入口申请待确认 | 有电脑或工具申请加入项目，通过审批后才会参与运行。 | 查看申请 |
| 数据过期 | 状态可能不是最新 | 最近同步失败，页面保留上一次可信状态。 | 刷新 |
| 无权限 | 只能查看部分项目 | 当前账号只能查看已授权项目，不能看到其他项目细节。 | 查看权限说明 |

## 创建项目入口

创建项目采用右侧抽屉或居中弹窗。入口文案是“创建项目”，不要写“Create Project”或内部对象名。

### 表单字段

| 字段 | 控件 | 说明 |
| --- | --- | --- |
| 项目名称 | 文本输入 | 用户看得懂的中文名称，必填。 |
| 项目负责人 | 人员选择 | 默认当前用户，可改为有权限的人。 |
| 项目目标 | 多行文本 | 简短说明项目要解决什么问题。 |
| 资料范围 | 多选 | 可选“飞书文档、代码仓库、会议纪要、手工上传资料”等中文项。 |
| 默认可见范围 | 单选 | “仅负责人可见 / 项目成员可见 / 按审批开放”。 |
| 需要接入电脑 | 开关 | 开启后提示下一步可邀请电脑。 |

### 权限、审批、确认、审计

1. 点击“创建项目”打开表单。
2. 表单底部展示“创建后会产生项目记录和审计记录，不会自动派发任务。”
3. 点击“提交申请”前弹出确认摘要：项目名称、负责人、资料范围、可见范围。
4. 服务端权限检查通过后，状态显示“项目创建申请已提交”或“项目已创建，等待资料接入”。
5. 权限不足时显示“你可以查看项目，但不能创建新项目。请联系项目负责人开通创建权限。”
6. 所有结果进入“审批与审计”，主界面可查看中文申请摘要。

### 空状态联动

当中枢没有项目时，主区域显示：

> 还没有项目。先创建一个项目，再邀请电脑接入，任务运行状态会在这里出现。

按钮：`创建项目`

## 邀请/注册电脑入口

入口文案为“邀请/注册电脑”。用户不需要理解 Runner。

### 注册流程

```txt
选择项目 -> 填电脑负责人和显示名 -> 选择可做工作 -> 查看授权摘要
-> 提交接入申请 -> 审批通过 -> 复制接入命令或打开接入说明
-> 电脑开始上报 -> 出现在电脑列表
```

### 表单字段

| 字段 | 控件 | 用户文案 |
| --- | --- | --- |
| 接入项目 | 下拉 | 这台电脑要服务哪个项目。 |
| 电脑负责人 | 人员选择 | 谁负责这台电脑。 |
| 电脑显示名 | 文本输入 | 例如“李雷的 MacBook Pro”。 |
| 可做工作 | 标签多选 | 设计、开发、测试、资料整理、项目管理、知识整理。 |
| 可用 Agent | 多选 | 产品经理 Agent、开发 Agent、测试 Agent 等中文角色。 |
| 可用模型 | 多选 | Codex、Claude、本地模型；具体模型名作为次级信息。 |
| 可用工具 | 多选 | 只显示已登记或可申请的工具中文名。 |
| 资料范围 | 多选 | 当前项目允许这台电脑读取的资料范围。 |

### 审批和确认

提交前展示确认摘要：

- 电脑：李雷的 MacBook Pro。
- 负责人：李雷。
- 服务项目：真知公司知识核心。
- 可做工作：开发、测试。
- 可用工具：代码检查、浏览器验证。
- 资料范围：当前项目文档和指定代码仓库。

确认文案：

> 通过后，这台电脑会向中枢上报运行状态。页面仍不会直接控制这台电脑执行任务。

审批中状态：

> 电脑接入申请已提交。审批通过前，这台电脑不会出现在可接任务列表。

审批通过后显示：

- “接入说明”按钮。
- “复制接入命令”按钮。
- “查看审批记录”按钮。

复制命令属于辅助动作，不改变中枢运行状态。复制按钮旁写明：

> 命令只用于这台电脑完成接入，不包含可见密钥明文。

## 工具注册入口

入口文案为“注册工具”。工具是公司能力资产，不是个人电脑里的私有脚本。

### 表单字段

| 字段 | 控件 | 说明 |
| --- | --- | --- |
| 工具名称 | 文本输入 | 用户可读名称，例如“网页截图检查”。 |
| 工具用途 | 多行文本 | 说明它能帮助 Agent 做什么。 |
| 工具负责人 | 人员选择 | 谁负责维护和授权。 |
| 适用项目 | 多选 | 可选全部项目或指定项目。 |
| 可使用 Agent | 多选 | 哪些 Agent 可调用。 |
| 风险等级 | 单选 | 只读查询、写入变更、外部网络、敏感资料。 |
| 凭据状态 | 单选/提示 | “已由负责人配置 / 需要负责人配置”。不填写密钥明文。 |
| 审批人 | 人员选择 | 高风险工具必须选择审批人。 |

### 工具状态

| 状态 | 主文案 | 说明 |
| --- | --- | --- |
| 待审批 | 等待工具审批 | 通过后才会出现在电脑可用工具里。 |
| 可用 | 已登记，可被授权使用 | 展示适用项目和负责人。 |
| 缺少凭据 | 需要负责人配置凭据 | 不展示 token、secret 或认证头。 |
| 已停用 | 工具已停用 | 历史任务仍可查看使用记录。 |
| 有风险 | 需要复核权限范围 | 展示风险原因和审批责任人。 |

工具使用情况在监管区只读展示，不能从任务详情里直接授权或禁用工具。

## 项目列表与项目任务明细

### 项目列表

项目列表用于回答“有哪些项目正在被哪些电脑处理”。

| 列 | 主文案示例 | 说明 |
| --- | --- | --- |
| 项目 | 真知公司知识核心 | 下方小字显示负责人，不显示内部项目 ID。 |
| 运行状态 | 运行中 / 暂无任务 / 有异常 | 中文状态标签。 |
| 电脑 | 3 台电脑参与 | 点击筛选该项目电脑。 |
| 任务 | 8 项任务，2 项待验收 | 点击进入任务明细。 |
| 最近更新 | 3 分钟前 | 来自中枢最近状态。 |
| 风险 | 1 项异常 | 无异常时显示“无明显风险”。 |

### 项目任务明细

任务明细采用可排序表格，首列是任务名称，第二列是对应需求。

| 列 | 示例 | 设计要求 |
| --- | --- | --- |
| 任务 | 完成工作台只读状态页 | 主标题用任务可读名。 |
| 对应需求 | 阶段二方案二中枢监管 | 需求标题可点击进入详情抽屉。 |
| 当前进展 | 执行中，约 60% | 不显示 raw status。 |
| 负责 Agent | 开发 Agent | 如果多个 Agent，显示“开发 Agent +1”。 |
| 执行电脑 | 李雷的 MacBook Pro | 用电脑显示名和负责人。 |
| 模型 | Codex / Claude Sonnet | 主文案先显示产品名，模型细项放次级。 |
| Token 消耗 | 输入 18.4k，输出 3.2k | 明确这是模型用量，不是凭据 token。 |
| 使用工具 | 代码检查、浏览器验证 | 超过 2 个折叠为“+2”。 |
| 最后更新 | 2 分钟前 | 超时则变为异常色。 |
| 验收状态 | 待验收 / 已通过 / 需修改 | 只读显示，不提供通过按钮。 |

任务详情抽屉分 5 段：

1. 任务摘要：任务、需求、当前进展、下一责任人。
2. 执行链路：负责 Agent、执行电脑、最近上报。
3. 模型和用量：模型、输入 token、输出 token、总 token。
4. 工具使用：工具名称、用途、最近调用结果。
5. 结果和证据：任务结果、检查项、验收状态、审计入口。

抽屉底部只放只读动作：

- 查看证据。
- 复制任务摘要。
- 查看审计记录。
- 展开技术详情。

## 任务与需求映射

任务与需求映射用“需求树 + 任务列表”的分栏视图。

```txt
左：需求
- 阶段二方案二中枢监管
  - 创建项目入口
  - 电脑注册入口
  - 工具注册入口
  - 运行监管只读

右：选中需求下的任务
任务名称 | 当前进展 | Agent | 电脑 | 模型 | Token | 工具 | 验收
```

状态表达：

- “需求下没有任务”：说明“这个需求还没有生成执行任务。”
- “任务没有对应需求”：说明“这项任务暂未挂到需求，可能需要项目经理整理。”
- “需求已完成但任务仍运行”：说明“需求状态和任务状态不一致，请查看审计记录。”

不展示需求内部编号为主列。技术详情里可显示脱敏标识。

## 电脑列表与电脑详情

### 电脑列表

电脑列表用于回答“哪台电脑在线、正在做什么、是否异常”。

| 列 | 示例 | 说明 |
| --- | --- | --- |
| 电脑 | 李雷的 MacBook Pro | 下方显示负责人和系统类型。 |
| 在线状态 | 在线 / 离线 / 响应慢 | 用最近心跳换算成人话。 |
| 当前项目 | 真知公司知识核心 | 多项目时显示“2 个项目”。 |
| 当前任务 | 工作台 UI 设计复核 | 无任务时显示“空闲”。 |
| Agent 状态 | 开发 Agent 正在工作 | 多 Agent 用标签。 |
| 使用模型 | Codex | Claude、本地模型作为同层标签。 |
| Token 消耗 | 今日 42.1k | 支持按今日/本任务切换。 |
| 工具 | 代码检查、浏览器验证 | 展示已使用工具，不代表授权编辑入口。 |
| 异常 | 无 / 心跳超时 | 有异常置顶。 |

### 电脑详情

电脑详情采用右侧抽屉或独立详情页。

1. 顶部摘要：电脑名称、负责人、在线状态、最近心跳、当前项目。
2. 当前工作：当前任务、对应需求、负责 Agent、进展摘要。
3. Agent 状态：在线、工作中、等待输入、上报失败、空闲。
4. 模型和 token：当前会话使用 Codex/Claude/本地模型，输入/输出/总 token。
5. 工具：已登记工具、最近调用、调用结果。
6. 异常：异常原因、影响任务、建议去哪里处理。
7. 证据与审计：上报记录、审批记录、操作记录、技术详情。

只读动作：

- 复制电脑状态摘要。
- 查看该电脑相关任务。
- 查看接入说明。
- 查看审计记录。

不提供“禁用电脑、释放任务、重试执行、修改授权”等运行控制按钮。若用户需要处理，文案引导：

> 请到电脑负责人所在会话或审批流程处理；这里保留中枢记录，不直接改运行状态。

## Agent 在线与工作状态

Agent 状态用中文标签，和电脑、任务同时出现。

| 状态 | 标签 | 说明 |
| --- | --- | --- |
| online | 在线 | Agent 最近有上报。 |
| working | 工作中 | Agent 正在处理任务。 |
| waiting_input | 等待输入 | 需要人工、审批或外部材料。 |
| reporting | 正在上报 | 已完成部分工作，正在回传结果。 |
| idle | 空闲 | 当前没有任务。 |
| offline | 离线 | 超过心跳阈值未上报。 |
| error | 异常 | 有明确失败原因。 |

主界面不得展示 raw enum。技术详情中如需排障，可显示“原始状态，已脱敏”。

## 模型、Token 和工具展示

### 模型

模型展示分两层：

- 主文案：Codex、Claude、本地模型。
- 次级信息：具体模型名、版本或运行方式。

示例：

```txt
模型：Codex
细节：GPT-5 Codex，云端会话
```

### Token

页面必须把“token 消耗”和“密钥/凭据 token”区分开。

推荐文案：

- “Token 消耗：输入 18.4k，输出 3.2k，总计 21.6k。”
- “凭据状态：已配置，不展示密钥内容。”

禁止文案：

- “原始模型凭据字符串：……”
- “原始租约凭据字段”
- “未脱敏密钥内容”

### 工具

工具展示用中文工具名、用途和最近结果。

```txt
使用工具：代码检查、浏览器验证、文档读取
最近结果：代码检查通过，浏览器验证有 1 项布局问题
```

工具详情默认不展示命令、路径、认证头或 stdout/stderr 原文。必要时进入“技术详情”，且需脱敏。

## 异常提示

异常必须说明四件事：发生了什么、影响什么、谁处理、下一步在哪里处理。

| 异常 | 标题 | 正文 | 动作 |
| --- | --- | --- | --- |
| 电脑离线 | 李雷的 Mac 已离线 | 这台电脑 18 分钟没有上报，正在影响“工作台 UI 设计复核”。 | 查看电脑详情 |
| 心跳慢 | 电脑响应变慢 | 最近 3 次心跳延迟，任务状态可能滞后。 | 查看上报记录 |
| 上报失败 | 进展没有成功回传 | 中枢还保留上一次可信状态，请到对应电脑检查。 | 查看异常说明 |
| 权限不足 | 电脑缺少项目授权 | 这台电脑不能读取当前项目资料，任务不会继续推进。 | 查看审批记录 |
| 工具缺失 | 缺少可用工具 | 当前任务需要“浏览器验证”，但电脑未获工具授权。 | 查看工具申请 |
| 结果缺证据 | 任务结果缺少证据 | 结果已回传，但检查项或证据不完整，暂不能验收。 | 查看证据 |
| 数据过期 | 中枢状态可能过期 | 最近同步失败，页面只展示上一次可信记录。 | 刷新 |

运行监管区的异常动作都是查看或跳转，不直接修复。

## 空状态

| 区域 | 标题 | 正文 | 入口 |
| --- | --- | --- | --- |
| 无项目 | 还没有项目 | 先创建项目，再接入电脑和工具。 | 创建项目 |
| 无电脑 | 还没有电脑接入 | 邀请同事电脑注册后，运行状态会出现在这里。 | 邀请/注册电脑 |
| 无工具 | 还没有登记工具 | 工具通过审批后，Agent 才能在任务中使用。 | 注册工具 |
| 项目无任务 | 这个项目还没有运行任务 | 有任务开始执行后，会显示 Agent、电脑、模型、token 和工具。 | 查看项目资料 |
| 电脑空闲 | 这台电脑当前空闲 | 它已接入中枢，但现在没有执行任务。 | 查看可做工作 |
| 无权限 | 当前只能查看部分内容 | 你没有查看完整项目或电脑详情的权限。 | 查看权限说明 |

空状态按钮可以进入入口操作区；如果用户无权限，按钮禁用并说明申请路径。

## 错误状态

| 场景 | 标题 | 正文 | 动作 |
| --- | --- | --- | --- |
| 中枢不可达 | 暂时连不上中枢 | 页面无法获取最新状态，没有改变任何项目或任务。 | 重试 |
| 数据格式错误 | 状态数据无法完整显示 | 部分字段不符合页面预期，已隐藏可能误导的信息。 | 查看技术详情 |
| 权限校验失败 | 无法确认你的查看权限 | 为避免误展示项目资料，页面暂时隐藏敏感内容。 | 重新登录 |
| 创建申请失败 | 项目申请没有提交成功 | 中枢没有创建项目，请检查权限后重试。 | 重试 |
| 电脑申请失败 | 电脑接入申请没有提交成功 | 这台电脑还不能接入中枢。 | 重试 |
| 工具申请失败 | 工具登记申请没有提交成功 | 工具不会出现在可用工具里。 | 重试 |

错误详情默认收起，主文案不展示堆栈、端点、内部 ID 或原始日志。

## 视觉语言

整体保持 Codex 风格：安静、清晰、信息密度合理，像状态工作台，不像营销页。

- 背景：浅灰白工作区，内容区白底或极浅色分隔。
- 字体层级：页面标题 20-24px，区块标题 14-16px，表格正文 13-14px。
- 卡片半径不超过 8px。
- 指标采用紧凑数字 + 中文标签，不用巨大装饰数字。
- 状态颜色克制：绿色可用、蓝色运行、黄色等待/审批、红色异常、灰色离线/只读。
- 不使用大面积渐变、装饰图形或营销式 hero。
- 表格和列表优先，卡片只用于总览指标、异常提示和详情摘要。
- 窄屏下改为顶部项目栏 + 分段标签 + 单列列表，详情抽屉全屏展示。

## 文案规则

主文案使用普通用户能懂的名词：

| 技术概念 | 主界面文案 |
| --- | --- |
| Runner | 电脑 / 执行入口 |
| runnerId | 电脑显示名 |
| ProjectTask | 任务 |
| Requirement | 需求 |
| AgentRun | 执行记录 |
| TaskResult | 任务结果 |
| Heartbeat | 最近上报 / 最近在线 |
| raw status | 中文业务状态 |
| token usage | Token 消耗 |
| credential token | 凭据状态 |
| capability | 可做工作 |
| tool registry | 工具登记 |

默认不在主界面出现这些内容：

- 内部英文 ID。
- 本地绝对路径。
- API endpoint。
- raw enum。
- token、secret、cookie、认证头。
- 错误堆栈和大段日志。

## 研发交接清单

- 顶部必须有统一项目选择，支持“全部项目”和单项目视图。
- 首页必须展示项目数、电脑数、运行中任务、异常电脑、待验收任务、待审批入口。
- 必须提供“创建项目”“邀请/注册电脑”“注册工具”三个入口。
- 三个入口必须接权限检查、审批状态、确认摘要和审计记录。
- 三个入口必须有幂等反馈：重复点击、网络重试或审批处理中时，页面显示同一条申请进度，不生成重复项目、重复电脑邀请或重复工具申请。
- 运行监管区必须只读，只有查看、筛选、展开、复制摘要、打开证据和查看审计。
- 项目任务明细必须展示任务、对应需求、进展、负责 Agent、执行电脑、模型、token 消耗、工具、最后更新时间和验收状态。
- 电脑详情必须展示在线状态、当前项目、当前任务、Agent、Codex/Claude/本地模型、工具、模型、token、最近心跳和异常。
- 异常、空状态、错误状态和数据过期必须有中文说明，不得只显示空白或 raw error。
- 主界面不得把内部 ID、路径、session、runner、device、token、endpoint 作为主文案。
- 技术详情必须默认收起，并做脱敏。

## 设计验收口径

产品经理复核时，普通用户应能回答：

- 现在中枢下有几个项目、几台电脑。
- 哪些项目有任务在运行，哪些任务待验收。
- 每个任务对应哪个需求，由哪个 Agent、哪台电脑处理。
- 任务使用了 Codex 还是 Claude，消耗了多少 token，用了哪些工具。
- 哪台电脑在线、离线、空闲或异常。
- 出现异常时，影响什么、谁负责、下一步去哪里处理。
- 哪些入口能发起申请，哪些区域只是查看运行事实。
- 为什么页面没有暴露内部 ID、路径、密钥或错误堆栈。
