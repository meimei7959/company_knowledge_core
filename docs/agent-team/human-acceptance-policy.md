# 人类验收策略

## 默认原则

低风险、内部、可回滚、证据充分的任务，可以由项目经理 Agent 自动放行。其余任务默认需要项目经理判断是否提交人类验收。

## 必须人审

以下事项必须人类验收或审批：

- 生产部署、线上配置、权限、密钥、安全策略；
- verified 知识、政策、工作流、跨团队标准；
- 对客户或外部团队的承诺；
- 高风险数据变更、删除、迁移；
- 需求范围、项目目标、交付标准发生重大变化；
- Agent 自评失败、证据不足或多次重试失败。
- 因上下文/token 预算、子 Agent 暂停或工具等待导致连续三次无法恢复，并可能影响交付承诺或验收判断。

## 项目经理可自动放行

同时满足以下条件时，项目经理 Agent 可以自动放行：

- 任务影响范围清楚；
- 证据和测试齐全；
- 无安全、权限、客户承诺和生产风险；
- 不改变 verified 知识、政策和跨团队流程；
- `qualityEvaluation.passed = true` 且 `commonRulesEvaluation.passed = true`。

## 通知要求

每个节点完成后必须通知项目经理 Agent。需要人类验收时，由项目经理决定通知谁、带哪些证据、是否暂停后续流转。

可恢复暂停不应通知为失败。项目经理 Agent 应通知当前等待状态、断点、预计恢复动作和下一次检查条件。
