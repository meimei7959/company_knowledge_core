# Phase 2：同一项目多电脑 PM 主控租约产品需求

## 一句话目标

同一项目同一时间只能有一个主控 PM Agent 持有有效调度租约；中枢只接受带有效租约的 PM 写入，拒绝其他写入并审计，工作台让用户看懂当前谁在主控、谁在协同、谁可接管，以及租约是否健康。

## 背景

Phase 2 已定义部署中枢、多设备 Runner、项目隔离、Runner 租约和工作台只读监管。真实多电脑协作后，还会出现另一层冲突：同一项目可能被多个项目经理 Agent 同时协调。

如果多个 PM 同时创建任务、改优先级、发调度指令、推进验收或恢复异常，中枢会出现重复派单、状态覆盖、接管不明、用户不知道该信谁的问题。Runner 任务租约只能保护单个任务领取，不能保证项目级调度意图唯一。

因此本期补充 PM 主控调度租约：项目级调度写入只能由当前主控 PM 发起；其他 PM 保持协同或备用身份，可查看、准备方案、等待接管，但不能静默改变项目调度状态。

## 产品原则

1. 单项目单主控：一个项目任一时刻最多一个有效 `pm_control_lease`。
2. 租约前置：所有 PM 发起的持久化写操作必须携带有效调度租约；无租约、过期租约、非主控租约、项目不匹配租约必须失败。
3. 协同可见：协同 PM 和备用 PM 能看到项目、主控、健康和接管路径，但不能绕过主控写入。
4. 接管显性：接管必须有原因、操作者、前任主控、后任主控、时间、租约变化和审计记录。
5. 审计可信：拒绝、续约、过期、释放、接管都必须写审计；审计不依赖 PM 自己声明。
6. 用户可懂：工作台首屏用姓名、Agent 名、电脑名、状态解释和下一步动作表达，不用内部 ID 作为主信息。

## 产品范围

### 本期必须有

- PM 主控调度租约生命周期：申请、持有、续约、释放、过期、接管、拒绝。
- PM 角色边界：主控 PM、协同 PM、备用 PM。
- PM 写操作租约校验：中枢在写入前校验项目、租约、持有人、状态、过期时间和 fencing token。
- 拒绝与审计：中枢拒绝无效写入，并写 `pm_control_lease.denied` 或等价审计事件。
- 工作台展示：主控 PM、协同 PM、备用 PM、租约健康、到期时间、最近心跳、接管记录、拒绝记录摘要。
- 用户可理解文案：冲突、无租约、租约过期、备用接管、写入被拒绝等场景必须有中文说明和下一步建议。
- 验收标准：覆盖单主控、协同/备用只读、无租约拒绝、接管、审计、工作台展示和兼容已有多设备 Runner。

### 本期明确不做

- 不让工作台直接派单、篡改 Runner 执行状态或改写 TaskResult。
- 不让协同 PM 静默获得调度写权限。
- 不把 Agent Ring 实现并入本仓库；本仓库只定义中枢、协议、审计、工作台和验收口径。
- 不用 PM 主控租约替代任务级 Runner 租约；两者同时存在，项目级租约保护调度意图，任务级租约保护具体执行。
- 不以内部 ID、路径、状态码作为用户主要解释。

## 用户与使用场景

| 用户 / Agent | 诉求 | 本需求提供 |
| --- | --- | --- |
| 项目 Owner | 知道当前哪个 PM 在管项目，冲突时能看懂为什么被拒绝。 | 主控卡片、租约健康、拒绝文案、接管记录。 |
| 主控 PM Agent | 独占项目级调度写入，避免其他 PM 覆盖自己的调度意图。 | `pm_control_lease`、续约、释放、fencing token。 |
| 协同 PM Agent | 查看项目、提出方案、准备接管，但不破坏主控状态。 | 协同身份、只读边界、接管准备入口。 |
| 备用 PM Agent | 主控失联或被释放后接管项目。 | 接管流程、接管原因、审计和新租约。 |
| 研发 / 测试 Agent | 明确对象、字段、API、工作台和验收口径。 | 产品输入、写操作清单、验收用例。 |

## 角色与权限边界

| 角色 | 定义 | 可做 | 不可做 |
| --- | --- | --- | --- |
| 主控 PM | 当前持有项目有效调度租约的 PM Agent。 | 创建/更新调度相关任务，调整优先级，推进验收路由，触发恢复流程，释放或续约租约。 | 绕过审计写入；用过期租约写入；跨项目使用租约。 |
| 协同 PM | 被授权参与项目但不持有主控调度租约的 PM Agent。 | 查看项目、查看任务、查看主控状态、准备建议、在获得主控合并或接管后写入。 | 直接创建/修改调度状态；覆盖主控决策；静默续约或替换主控。 |
| 备用 PM | 被配置为可在主控异常时接管的 PM Agent。 | 查看项目、查看健康、准备接管、在接管流程成功后成为新主控。 | 主控健康时无原因抢占；未生成新租约前写项目调度状态。 |

说明：所有由 PM Agent 发起、会被中枢持久化的项目调度写操作都必须带有效 `pm_control_lease`。协同 PM 的建议可先作为本地草稿或由主控 PM 带租约合并；中枢不得接受无租约的 PM 持久化写入。

## PM 写操作租约要求

### 必须带租约的写操作

| 写操作 | 示例 | 租约要求 |
| --- | --- | --- |
| 项目调度计划变更 | 更新项目阶段、调度策略、任务拆分方案。 | 当前项目有效主控租约。 |
| ProjectTask / KnowledgeTask 创建或更新 | 创建任务、改优先级、改 assignee、改 workflow、改阻塞状态。 | 当前项目有效主控租约。 |
| 调度指令 | 暴露任务、暂停调度、恢复调度、重试、取消、切换候选 Runner。 | 当前项目有效主控租约。 |
| 验收路由 | 指定验收人、推进等待验收、创建补验任务。 | 当前项目有效主控租约。 |
| PM 交接结果写入 | 释放主控、确认交接、写接管后的调度摘要。 | 当前项目有效主控租约；接管动作本身走接管流程。 |
| 用户通知型调度写入 | 通知任务改派、阻塞恢复、需要人工审批。 | 当前项目有效主控租约，并写 NotificationRecord。 |
| 工作台 PM 写入口 | 页面按钮触发的项目级 PM 写动作。 | 前端必须提交租约；后端仍以中枢校验为准。 |

### 不属于 PM 调度写入但仍需审计

| 操作 | 产品口径 |
| --- | --- |
| 中枢自动写 AuditLog | 系统可信记录，不需要 PM 租约，但必须关联触发请求。 |
| Runner heartbeat / progress_update / result_writeback | 继续按 Runner 租约和 Agent Ring 协议校验，不被 PM 主控租约替代。 |
| 用户只读查看 | 不需要 PM 租约，但仍受项目权限和脱敏规则限制。 |
| 接管申请 / 接管审批 | 属于租约管理流程；成功前不得改变项目调度状态，必须写接管审计。 |

## 中枢校验与拒绝

中枢在任何 PM 写入口落库前必须校验：

1. 请求包含 `projectId`、`pmAgentId`、`leaseId`、`fencingToken`。
2. 租约存在，属于同一项目。
3. 租约状态为 `active` 或策略允许的 `expiring`。
4. 请求 PM 是租约持有人。
5. 当前时间未超过 `expiresAt`。
6. `fencingToken` 等于当前最新 token，旧 token 不能写入。
7. 写操作类型在主控 PM 允许范围内。

拒绝时必须：

- 不写目标业务对象。
- 写 `AuditLog`，事件名为 `pm_control_lease.denied` 或等价事件。
- 审计至少包含项目、请求 PM、当前主控 PM、请求动作、拒绝原因、请求来源、时间、关联租约。
- 返回用户可理解说明和下一步建议。

## 接管流程

### 主控健康时

1. 备用 PM 或用户发起接管意图，填写原因。
2. 中枢判断当前租约健康，提示“当前主控仍在线，接管会中断其调度权”。
3. 需要项目 Owner 或具备权限者确认。
4. 确认后旧租约标记 `taken_over`，生成新租约和接管记录。
5. 旧主控再写入时因 fencing token 失效被拒绝并审计。

### 主控过期或失联时

1. 中枢将租约标记为 `stale` 或 `expired`。
2. 工作台提示可接管，并展示最后心跳、到期时间、候选备用 PM。
3. 备用 PM 发起接管，必须填写原因。
4. 中枢生成新租约、接管记录和审计。
5. 项目调度入口恢复为新主控可写。

### 主控主动释放

1. 主控 PM 释放租约，必须带当前有效租约。
2. 中枢标记 `released`，写审计。
3. 若存在备用 PM，工作台提示“等待备用 PM 接管”；若无备用，提示“暂无主控 PM，项目调度写入暂停”。

## 工作台需求

### 项目详情：PM 主控卡片

| 信息 | 展示文案要求 |
| --- | --- |
| 主控 PM | 显示 Agent 名称、Owner、所在电脑，不以 ID 为主。 |
| 租约健康 | `健康`、`即将到期`、`心跳延迟`、`已过期`、`已接管`。 |
| 最近续约 | “最近 20 秒前续约”，而不是裸时间戳。 |
| 到期时间 | “预计 14:35 到期，还剩 2 分钟”。 |
| 当前写入状态 | “只有当前主控 PM 可以改项目调度”。 |
| 下一步动作 | 健康时展示查看详情；过期时展示接管；无主控时展示选择备用 PM。 |

### 协同 PM 与备用 PM 列表

| 信息 | 展示要求 |
| --- | --- |
| 协同 PM | Agent 名、在线状态、最近活跃、可参与范围。 |
| 备用 PM | Agent 名、接管优先级、接管条件、最近活跃。 |
| 权限解释 | 明确“可查看和准备方案，不能直接改调度”。 |
| 异常提示 | 备用 PM 离线、无备用 PM、备用无项目权限时必须提示。 |

### 租约健康解释

| 状态 | 人能读懂的含义 | 下一步 |
| --- | --- | --- |
| healthy | 主控 PM 在线并按时续约。 | 继续观察。 |
| expiring | 租约快到期，最近续约仍正常。 | 提醒主控续约或准备备用。 |
| stale | 主控 PM 心跳延迟，可能失联。 | 展示最后心跳和接管入口。 |
| expired | 租约已过期，中枢暂停接受旧主控写入。 | 备用 PM 或 Owner 接管。 |
| taken_over | 项目已由新 PM 接管，旧租约失效。 | 查看接管记录。 |
| released | 主控主动释放，当前无主控。 | 选择备用 PM 接管。 |

### 接管记录

工作台必须展示接管时间线：

- 接管时间。
- 前任主控 PM。
- 后任主控 PM。
- 触发者。
- 原因。
- 接管前租约状态。
- 新租约健康。
- 审计入口。

### 拒绝记录摘要

项目详情可展示最近拒绝记录：

| 展示字段 | 示例 |
| --- | --- |
| 时间 | “3 分钟前”。 |
| 请求方 | “协同 PM：需求拆分 PM”。 |
| 动作 | “尝试更新任务优先级”。 |
| 拒绝原因 | “不是当前主控 PM”。 |
| 建议 | “联系主控 PM 合并，或在主控失联后发起接管”。 |

## 用户可理解文案

| 场景 | 文案 |
| --- | --- |
| 主控健康 | “当前由 PM A 主控此项目，租约健康。其他 PM 可以查看和准备建议，但不能直接改项目调度。” |
| 无主控 | “此项目暂无主控 PM。为避免多人同时修改，中枢暂不接受 PM 调度写入。请选择一个备用 PM 接管。” |
| 写入无租约 | “写入被拒绝：请求没有有效主控租约。请由当前主控 PM 操作，或先完成接管。” |
| 非主控写入 | “写入被拒绝：你不是当前主控 PM。当前主控是 PM A。” |
| 租约过期 | “写入被拒绝：主控租约已过期。请续约或让备用 PM 接管。” |
| 项目不匹配 | “写入被拒绝：此租约不属于当前项目。请重新进入项目后操作。” |
| 旧主控写入 | “写入被拒绝：项目已由 PM B 接管，旧主控租约已失效。” |
| 接管成功 | “PM B 已接管项目。PM A 的旧租约已失效，后续调度写入由 PM B 负责。” |
| 接管需确认 | “当前主控仍在线。接管会中断其调度权，请确认原因和影响范围。” |

## 指标与告警

| 指标 | 用途 |
| --- | --- |
| active_pm_control_lease_count_by_project | 发现同项目多主控异常，目标恒为 0 或 1。 |
| pm_control_lease_denied_write_count | 观察冲突写入、权限配置或前端漏传租约。 |
| pm_control_lease_takeover_count | 观察主控稳定性和接管频率。 |
| pm_control_lease_stale_duration | 衡量主控失联恢复速度。 |
| pm_control_lease_renew_latency | 判断续约链路健康。 |
| pm_control_lease_workbench_visibility_error | 工作台展示与中枢状态不一致时告警。 |

告警触发：

1. 同一项目出现超过一个 `active` PM 控制租约。
2. 主控租约过期超过阈值但无备用接管。
3. 无租约写入拒绝在短时间内持续上升。
4. 工作台 read model 显示主控与中枢最新租约不一致。

## 交付给架构、研发与测试的产品输入

架构 Agent 需要补齐：

- `PmControlLease`、`ProjectPmParticipant`、`PmLeaseTakeoverRecord` 数据模型。
- PM 写入口租约校验中间层。
- acquire / renew / release / takeover / status 查询 API 或 CLI。
- 工作台 read model 聚合方案。
- 拒绝审计事件和异常恢复方案。

研发 Agent 需要实现：

- core/server/cli/workbench/tests 的最小闭环。
- 所有 PM 写入口的租约校验。
- 拒绝不落业务对象、只落审计。
- 工作台中文展示，不以内部 ID 做主信息。
- 与已有 Runner 租约、项目隔离、权限边界兼容。

测试 Agent 需要验证：

- 单主控。
- 协同 / 备用只读。
- 无租约、过期租约、非主控租约、项目不匹配租约拒绝。
- 接管生成新租约并让旧 token 失效。
- 审计完整。
- 工作台展示完整且用户可理解。
- 已有多设备 Runner 能力不回归。

## 验收标准

### 功能验收

| 用例 | 操作 | 期望结果 |
| --- | --- | --- |
| 单主控获取 | 两个 PM 同时为同一项目申请主控租约。 | 只有一个 PM 获得 `active` 租约；另一个看到已被占用的可读提示；审计可查。 |
| 主控写入成功 | 当前主控 PM 带有效租约创建或更新项目任务。 | 写入成功，业务对象关联租约，写审计。 |
| 无租约拒绝 | PM 不带租约调用项目调度写入口。 | 写入失败，不产生目标业务变更，写 `pm_control_lease.denied` 审计。 |
| 协同 PM 拒绝 | 协同 PM 尝试改任务优先级或调度状态。 | 写入失败，提示“你不是当前主控 PM”，审计记录当前主控和请求方。 |
| 备用 PM 拒绝 | 备用 PM 在未接管前尝试写项目调度状态。 | 写入失败，提示需先接管，审计可查。 |
| 过期租约拒绝 | 原主控 PM 使用过期租约写入。 | 写入失败，提示租约过期，业务对象不变，审计可查。 |
| 项目不匹配拒绝 | PM 使用项目 A 租约写项目 B。 | 写入失败，提示租约不属于当前项目，项目 B 不受污染。 |
| 旧 token 拒绝 | 接管后旧主控继续用旧 fencing token 写入。 | 写入失败，提示项目已接管，审计记录旧主控写入被拒。 |
| 主控过期接管 | 主控停止续约直到租约过期，备用 PM 发起接管。 | 生成新租约、接管记录和审计；新主控可写，旧主控不可写。 |
| 主控健康接管 | 主控健康时用户强制接管。 | 必须填写原因并确认；成功后旧租约失效，工作台展示接管记录。 |
| 主控释放 | 主控 PM 主动释放租约。 | 租约变为 `released`，项目进入无主控状态或等待备用接管。 |
| 工作台展示 | 打开项目详情。 | 看到主控 PM、协同 PM、备用 PM、租约健康、到期时间、最近心跳、接管记录。 |
| 用户文案 | 触发无租约、非主控、过期、接管成功等状态。 | 页面或响应显示中文可理解说明和下一步建议，不以内部 ID 为主。 |
| Runner 兼容 | 运行已有多设备 Runner 领取任务、上报进展、写回结果流程。 | 任务级 Runner 租约逻辑不回归；PM 主控租约不替代 Runner 租约。 |

### 通过标准

1. 同一项目数据库或 read model 中不能同时出现两个 `active` PM 主控租约。
2. 所有 PM 调度写入口都能证明先校验租约，再写业务对象。
3. 所有拒绝写入都不产生目标业务对象变化，并产生审计。
4. 接管记录可从工作台追溯到审计。
5. 工作台主信息为人能理解的名称、状态和建议，不要求用户先理解内部 ID。
6. 单机、本地测试、多设备 Runner 既有验收不回归。
